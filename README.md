# Прототип цифрового двойника рынка краткосрочной аренды автомобилей Санкт-Петербурга

Исследовательский проект для сценарного моделирования спроса, доступности автомобилей и перебалансировки парка в системе краткосрочной аренды автомобилей Санкт-Петербурга.

Проект развивает результаты научной работы предыдущего семестра: ранее был сформирован спросовой слой городской мобильности, а в данной работе на его основе построена модель функционирования сервиса краткосрочной аренды автомобилей.

---

## Описание

Проект реализует прототип цифрового двойника, который позволяет оценивать работу сервиса краткосрочной аренды автомобилей в разных сценариях:

* базовый режим работы;
* рост спроса;
* нехватка доступного парка;
* высокий спрос при нехватке автомобилей;
* перебалансировка автомобилей;
* перебалансировка при повышенной нагрузке.

Модель рассчитывает ключевые показатели качества обслуживания и формирует интерактивную карту пространственного распределения потерянного спроса.

---

## Возможности

Проект позволяет:

* использовать спросовой слой, подготовленный в научной работе предыдущего семестра;
* моделировать поток пользовательских заявок;
* учитывать доступность автомобилей по расчётным зонам;
* сравнивать несколько сценариев работы сервиса;
* рассчитывать выполненные заявки, потерянный спрос и долю выполненных заявок;
* оценивать эффект перебалансировки автомобилей;
* строить итоговые таблицы, графики и интерактивную карту.

---

## Как устроена модель

Общая логика проекта:

```text
Спросовой слой предыдущего исследования
        ↓
Расчётные зоны Санкт-Петербурга
        ↓
Модельные заявки пользователей
        ↓
Доступные автомобили
        ↓
Правила обслуживания заявки
        ↓
Сценарии изменения спроса и парка
        ↓
Метрики, графики и интерактивная карта
```

Проект переводит подготовленный спросовой слой в сценарную модель работы сервиса: для каждой заявки проверяется наличие автомобиля, фиксируется результат обслуживания и рассчитываются итоговые показатели.

---

## Сценарии

| Сценарий               | Назначение                               |
| ---------------------- | ---------------------------------------- |
| `baseline`             | базовый режим работы модели              |
| `high_demand`          | рост числа заявок                        |
| `fleet_shortage_clean` | нехватка доступных автомобилей           |
| `system_stress`        | высокий спрос и нехватка парка           |
| `simple_relocation`    | перебалансировка автомобилей             |
| `relocation_stress`    | перебалансировка при повышенной нагрузке |

Сценарии позволяют сравнить, как меняется работа сервиса при изменении спроса, доступности автомобилей и пространственного распределения парка.

---

## Основные показатели

| Показатель                             | Описание                                                |
| -------------------------------------- | ------------------------------------------------------- |
| `total_orders`                         | общее число заявок                                      |
| `completed_orders`                     | число выполненных заявок                                |
| `lost_demand_no_vehicle`               | потерянный спрос из-за отсутствия доступного автомобиля |
| `client_rejection_after_vehicle_found` | отказ после нахождения автомобиля                       |
| `completion_rate`                      | доля выполненных заявок                                 |
| `lost_demand_no_vehicle_rate`          | доля потерянного спроса                                 |
| `avg_distance_to_vehicle_km`           | среднее расстояние до автомобиля                        |
| `avg_fleet_utilization`                | средняя загрузка парка                                  |
| `relocated_vehicles`                   | число перебалансированных автомобилей                   |

## Итоговые результаты

Финальная таблица сценарного моделирования сохраняется в файле:

```text
outputs/final_scenario_comparison.csv
```

Ключевые результаты финального запуска:

| Сценарий                       | Заявки | Выполнено | Потерянный спрос | Доля выполненных заявок |
| ------------------------------ | -----: | --------: | ---------------: | ----------------------: |
| Базовый режим                  |   1151 |       397 |               73 |                  34,5 % |
| Рост спроса                    |   1672 |       634 |               90 |                  37,9 % |
| Нехватка парка                 |   1151 |        74 |             1052 |                   6,4 % |
| Высокий спрос и нехватка парка |   1828 |        99 |             1696 |                   5,4 % |
| Простая перебалансировка       |   1151 |       415 |               12 |                  36,1 % |
| Перебалансировка при нагрузке  |   1672 |       607 |               31 |                  36,3 % |

Основной результат: сценарии нехватки парка приводят к резкому росту потерянного спроса, а перебалансировка автомобилей существенно снижает дефицит.

## Быстрый запуск

### 1. Перейти в папку проекта

```bat
cd C:\Users\Александра\Desktop\НИР2\mvp_v12_final_spb_work
```

### 2. Создать виртуальное окружение

```bat
python -m venv .venv
```

### 3. Активировать окружение

```bat
.venv\Scripts\activate.bat
```

После активации в терминале должно появиться:

```text
(.venv)
```

### 4. Установить зависимости

```bat
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

При необходимости можно дополнительно установить основные библиотеки:

```bat
python -m pip install pandas numpy geopandas folium branca matplotlib pytest
```

### 5. Запустить проверку

```bat
python -m pytest -q
```

### 6. Запустить полный сценарий моделирования

```bat
python experiments\run_final_pipeline.py
```

Или напрямую через интерпретатор виртуального окружения:

```bat
.venv\Scripts\python.exe experiments\run_final_pipeline.py
```

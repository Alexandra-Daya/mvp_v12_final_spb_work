# Прототип цифрового двойника рынка краткосрочной аренды автомобилей Санкт-Петербурга

Репозиторий содержит исследовательский прототип цифрового двойника рынка краткосрочной аренды автомобилей Санкт-Петербурга. Проект предназначен для сценарного моделирования спроса, доступности автомобилей, возникновения потерянного спроса и эффекта перебалансировки парка.

Работа является продолжением предыдущей научной работы по прогнозированию транспортного спроса. В качестве входной основы используются восстановленные расчётные зоны, матрица корреспонденций и территориальные признаки. В текущей работе к этому слою добавлена имитационная модель функционирования сервиса: заявки пользователей, доступные автомобили, правила назначения автомобиля, отказы, потерянный спрос и сценарное сравнение.

## Назначение проекта

Цель проекта — разработать воспроизводимый прототип, позволяющий оценивать, как меняется качество обслуживания при разных условиях спроса и предложения.

Модель позволяет анализировать:

* распределение спроса по расчётным зонам;
* вероятность отсутствия доступного автомобиля в зоне;
* долю выполненных заявок;
* потерянный спрос из-за нехватки автомобилей;
* эффект перебалансировки парка;
* пространственное расположение зон дефицита на интерактивной карте.

Проект не претендует на точное воспроизведение работы конкретного оператора. Он является исследовательской моделью, построенной на восстановленных и открытых данных.

## Общая логика работы

Последовательность расчёта включает несколько этапов:

1. загрузка расчётных зон и восстановленного спроса, полученных в предыдущей научной работе по прогнозированию транспортного спроса;
2. формирование модельных заявок пользователей;
3. распределение доступных автомобилей по зонам;
4. выполнение сценарных экспериментов;
5. расчёт итоговых показателей;
6. построение интерактивной карты;
7. сохранение таблиц, диагностических файлов и отчётных материалов.

Схематично логика проекта выглядит так:

```text
Предыдущая научная работа по прогнозированию транспортного спроса
    ↓
Расчётные зоны и восстановленный спрос
    ↓
Имитационная модель сервиса краткосрочной аренды автомобилей
    ↓
Сценарии спроса, парка и перебалансировки
    ↓
Таблицы, показатели и интерактивная карта
```

## Сценарии моделирования

В проекте реализованы следующие сценарии:

| Сценарий                       | Смысл                                                                       |
| ------------------------------ | --------------------------------------------------------------------------- |
| Базовый режим                  | работа системы при исходных условиях                                        |
| Рост спроса                    | проверка устойчивости при увеличении числа заявок                           |
| Нехватка парка                 | оценка последствий недостатка автомобилей                                   |
| Высокий спрос и нехватка парка | стрессовый режим с одновременным ростом спроса и снижением доступного парка |
| Простая перебалансировка       | перенос части автомобилей в зоны ожидаемого дефицита                        |
| Перебалансировка при нагрузке  | проверка эффекта управления парком в более тяжёлых условиях                 |

## Основные показатели

В результате расчётов формируются следующие показатели:

| Показатель                       | Интерпретация                                                        |
| -------------------------------- | -------------------------------------------------------------------- |
| Общее число заявок               | количество модельных пользовательских обращений в сценарии           |
| Выполненные заявки               | число заявок, для которых был найден доступный автомобиль            |
| Потерянный спрос                 | заявки, не выполненные из-за отсутствия доступного автомобиля        |
| Отказ клиента                    | отказ после найденного автомобиля, связанный с условиями доступности |
| Доля выполненных заявок          | отношение выполненных заявок к общему числу заявок                   |
| Доля потерянного спроса          | отношение потерянного спроса к общему числу заявок                   |
| Расчётная выручка                | модельная оценка результата выполненных поездок                      |
| Среднее расстояние до автомобиля | расстояние от пользователя до назначенного автомобиля                |
| Перемещённые автомобили          | число автомобилей, перераспределённых при перебалансировке           |
| Максимальный дефицит по зонам    | наибольшее значение дефицита в отдельной расчётной зоне              |

## Установка и запуск

Перед запуском рекомендуется использовать виртуальное окружение.

### 1. Перейти в папку проекта

```bat
cd mvp_v12_final_spb_work
```

### 2. Создать виртуальное окружение

```bat
python -m venv .venv
```

### 3. Активировать виртуальное окружение

```bat
.venv\Scripts\activate.bat
```

### 4. Установить зависимости

```bat
python -m pip install -r requirements.txt
```

### 5. Проверить проект тестами

```bat
python -m pytest -q
```

Ожидаемый результат: тесты должны завершиться без ошибок.

### 6. Запустить итоговую последовательность расчёта

```bat
python experiments\run_final_pipeline.py
```

После успешного запуска будут созданы итоговые таблицы, диагностические файлы и интерактивная карта.

## Интерпретация карты

Интерактивная карта показывает пространственную концентрацию потерянного спроса по сценариям. Тёплые области отражают зоны, где в рамках модели чаще возникает ситуация отсутствия доступного автомобиля.

Важно учитывать:

* расчётные зоны являются модельными, а не административными районами;
* сетка зон унаследована из предыдущей научной работы по прогнозированию транспортного спроса;
* зоны, частично попадающие на воду, не должны интерпретироваться как реальные районы спроса;
* тепловой слой показывает относительную концентрацию внутри выбранного сценария;
* абсолютные значения необходимо смотреть в итоговых таблицах.

## Ключевые результаты

Моделирование показывает, что нехватка автомобилей резко увеличивает потерянный спрос и снижает долю выполненных заявок. Простая перебалансировка способна существенно уменьшить число заявок, не выполненных из-за отсутствия доступного автомобиля, особенно в стрессовых сценариях.

Итоговые значения по сценариям:

| Сценарий                       | Заявки | Выполнено | Потерянный спрос | Доля выполненных заявок |
| ------------------------------ | -----: | --------: | ---------------: | ----------------------: |
| Базовый режим                  |   1151 |       397 |               73 |                   34,5% |
| Рост спроса                    |   1672 |       634 |               90 |                   37,9% |
| Нехватка парка                 |   1151 |        74 |             1052 |                    6,4% |
| Высокий спрос и нехватка парка |   1828 |        99 |             1696 |                    5,4% |
| Простая перебалансировка       |   1151 |       415 |               12 |                   36,1% |
| Перебалансировка при нагрузке  |   1672 |       607 |               31 |                   36,3% |

Эти значения следует рассматривать как результаты исследовательского сценарного эксперимента, а не как статистику реального оператора.

## Основной вывод

Результаты показывают, что доступность автомобилей является критическим фактором качества обслуживания. При недостатке парка система быстро переходит в режим высокого потерянного спроса. Перебалансировка автомобилей позволяет снизить этот эффект и повысить устойчивость сервиса, однако в данной версии она реализована как исследовательская эвристика, а не как оптимизационная модель реального оператора.

## Ограничения

Проект имеет несколько принципиальных ограничений:

1. используются восстановленные данные предыдущей научной работы по прогнозированию транспортного спроса, а не закрытые журналы операторов;
2. модельные заявки не являются реальными заказами пользователей;
3. координаты автомобилей, тарифы и фактические бронирования операторов не используются;
4. расчётные зоны не совпадают с административными районами;
5. правила поведения пользователей и автомобилей заданы модельно;
6. перебалансировка реализована как исследовательская эвристика, а не как оптимизационная модель реального оператора.

Эти ограничения явно учитываются при интерпретации результатов.

## Научная и практическая значимость

Научная значимость проекта состоит в переходе от восстановленного территориального спроса, полученного в предыдущей научной работе по прогнозированию транспортного спроса, к сценарному цифровому двойнику сервиса краткосрочной аренды автомобилей.

Практическая значимость заключается в возможности предварительно оценивать последствия управленческих решений:

* увеличение или сокращение парка;
* изменение спроса;
* появление зон дефицита;
* применение перебалансировки автомобилей;
* сравнение сценариев до внедрения решений в реальной системе.
